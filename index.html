Structured Query Language [SQL]

Structured query language (SQL) is a programming language for storing and processing information in a relational database. When data is organized like a table, with rows and columns, SQL helps us do things like adding, updating, finding, and sorting that data. It's like the handy tool that makes sure all the information is well-organized and easy to work with, kind of like a digital librarian for data!

Foundation of SQL

In the 1970s, at IBM's San Jose Research Laboratory, two clever minds, Donald and Raymond, were on a mission. They were like digital pioneers tasked with creating a language to talk to and organize databases. Picture it like building a language for computers to manage information neatly.

Their creation was initially called SEQUEL (Structured English Query Language). It was the magical tool that made handling data in tables so much easier. But, uh-oh, there was a little hiccup â€“ trademark issues. So, they gave it a new name: SQL.

Now, SQL became a big shot, a star even. It became the standard language for talking to databases in a way that everyone understood. Thanks to Donald and Raymond's work, organizing and chatting with databases became a breeze. And that's how our data lived happily ever after!

Types of SQL Commands

1. Data Definition Language (DDL)

Used for defining and managing all database objects (tables, indexes, etc.).

2. Data Manipulation Language (DML)

Used for managing data within schema objects (insert, update, delete).

3. Data Control Language (DCL)

Used to control access to data (grant, revoke).

4. Transaction Control Commands (TCL)

Used for managing transactions (commit, rollback).

5. Data Query Language (DQL)

Used to query data from the database (select).

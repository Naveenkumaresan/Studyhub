Data Definition Language (DDL)

Data Definition Language (DDL) in SQL is used to define and manage the structure of database objects. It allows users to create, modify, and delete database objects such as tables, indexes, views, and schemas. DDL commands are used to define the schema and structure of the database, including its tables and their columns, constraints, indexes, and other properties.

Commands

CREATE

The CREATE command is used to create new database objects such as tables, indexes, views, schemas, and more. For example, you can use the CREATE TABLE command to create a new table with specified columns and data types.

ALTER

The ALTER command is used to modify existing database objects. It allows you to add, modify, or drop columns, constraints, indexes, and other properties of tables, views, or other objects.

DROP

The DROP command is used to delete existing database objects such as tables, views, indexes, and schemas. Be cautious when using DROP commands as they permanently delete the specified objects and their data.

TRUNCATE

The TRUNCATE command is used to remove all records from a table without deleting the table itself. It is faster than the DELETE command because it does not generate individual delete operations for each row.
